# 部署流程

## 11.1 部署概述

Economic Adventure 支持多平台部署，主要包括：

| 平台 | 部署方式 | 适用场景 |
|------|---------|---------|
| Web 浏览器 | 直接部署 HTML | 网页版测试、H5 推广 |
| 微信小程序 | 微信开发者工具上传 | 主要发布平台 |
| 其他小游戏平台 | LayaAir 构建导出 | 渠道扩展 |

## 11.2 Web 部署

### 11.2.1 构建项目

1. 打开 LayaAir IDE
2. 选择 `发布 -> 发布项目`
3. 选择 `Web` 平台
4. 配置构建选项：
   - 输出路径：`bin/`
   - 压缩代码：建议开启
   - 合并文件：可选
5. 点击 `发布`

### 11.2.2 部署到服务器

构建完成后，`bin/` 目录包含以下文件：

```
bin/
├── index.html          # 入口页面
├── js/
│   ├── bundle.js       # 游戏主代码
│   └── index.js        # 入口脚本
├── libs/
│   └── laya.xxx.js     # 引擎库文件
└── res/                # 游戏资源（图片、图集等）
```

**部署步骤**：

1. 将 `bin/` 目录下所有文件上传到 Web 服务器
2. 确保服务器支持静态资源访问
3. 配置 MIME 类型（如需要）

**Nginx 配置示例**：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/economic-adventure;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1M;
        add_header Cache-Control "public, immutable";
    }
}
```

## 11.3 微信小程序部署

### 11.3.1 构建项目

1. 打开 LayaAir IDE
2. 选择 `发布 -> 发布项目`
3. 选择 `微信小游戏` 平台
4. 配置构建选项：
   - 输出路径：指定导出目录
   - 小游戏适配：启用
5. 点击 `发布`

### 11.3.2 导入微信开发者工具

1. 打开微信开发者工具
2. 选择 `导入项目`
3. 选择构建输出的目录
4. 输入 AppID：`wx47483884ad0a7a56`
5. 点击 `导入`

### 11.3.3 预览与调试

1. 在微信开发者工具中点击 `编译`
2. 使用模拟器预览效果
3. 点击 `真机调试` 扫码在手机上测试
4. 查看控制台输出，排查问题

### 11.3.4 上传发布

1. 确认版本功能正常
2. 点击 `上传` 按钮
3. 填写版本号和项目备注
4. 上传成功后，登录微信小程序后台
5. 进入 `版本管理`，将开发版本提交审核
6. 审核通过后，点击 `发布`

## 11.4 版本管理

### 11.4.1 版本号规范

采用语义化版本控制（SemVer）：

```
主版本号.次版本号.修订号
```

| 版本类型 | 说明 | 示例 |
|---------|------|------|
| 主版本号 | 重大更新，不兼容旧版本 | 1.0.0 -> 2.0.0 |
| 次版本号 | 功能更新，向下兼容 | 1.0.0 -> 1.1.0 |
| 修订号 | Bug 修复 | 1.0.0 -> 1.0.1 |

### 11.4.2 发布检查清单

- [ ] 代码审查完成
- [ ] 本地测试通过
- [ ] 真机测试通过
- [ ] 版本号已更新
- [ ] 更新日志已编写
- [ ] 构建产物验证通过
- [ ] 资源文件无遗漏

## 11.5 持续集成（可选）

### 11.5.1 GitLab CI 配置

项目已包含 GitLab CI 配置模板，可在 `.gitlab-ci.yml` 中配置：

```yaml
stages:
  - build
  - deploy

build:
  stage: build
  script:
    - npm install
    - npm run build
  artifacts:
    paths:
      - bin/

deploy:
  stage: deploy
  script:
    - echo "Deploy to server"
  only:
    - main
```

## 11.6 回滚策略

如发布版本出现问题，可按以下步骤回滚：

1. **Web 部署**：直接替换为上一版本的构建产物
2. **微信小程序**：
   - 登录微信小程序后台
   - 进入 `版本管理`
   - 将线上版本回退到上一版本

## 11.7 性能优化建议

### 11.7.1 资源优化

- 使用图集（Atlas）合并小图
- 压缩 PNG 图片
- 按需加载资源

### 11.7.2 代码优化

- 启用代码压缩（minified）
- 合并 JavaScript 文件
- 移除未使用的代码

### 11.7.3 加载优化

- 配置合理的加载进度条
- 分阶段加载资源
- 使用 CDN 加速静态资源
