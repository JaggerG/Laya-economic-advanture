# 数据库设计

## 9.1 设计说明

Economic Adventure 当前版本采用**内存数据存储**方案，所有游戏数据以 TypeScript 对象形式保存在运行时内存中。数据定义分布在两个核心文件中：

- **配置数据**：`src/model/index.ts` — 游戏静态配置
- **玩家数据**：`src/model/userInfo.ts` — 玩家动态数据

> **注意**：当前版本未接入持久化存储（如本地存储、服务器数据库）。后续版本如需实现存档功能，可参考以下数据结构设计。

## 9.2 配置数据（Static Config）

### 9.2.1 资产配置表（assets）

```json
{
  "Assets": [
    {
      "name": "Potato",
      "amount": 0,
      "img_url": "",
      "child": [
        {
          "name": "Farmer",
          "amount": 250,
          "level": 1,
          "outcome": 10,
          "produce": "Parent",
          "time": 4,
          "desc": "produce potatoes",
          "cost": [
            { "name": "Parent", "quantity": 2 },
            { "name": "Employee", "quantity": 1 }
          ],
          "bonus": {
            "Power": { "name": "POWER", "quantity": 1, "url_img": "" },
            "Discount": { "name": "DISCOUNT", "quantity": 1, "url_img": "" },
            "Speed": { "name": "SPEED", "quantity": 1, "url_img": "" },
            "Chance": { "name": "CHANCE", "quantity": 1, "url_img": "" },
            "Bonus": { "name": "BONUS", "quantity": 1, "url_img": "" }
          }
        }
      ]
    }
  ]
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| name | string | 资源名称（唯一标识） |
| amount | number | 初始数量 |
| img_url | string | 资源图片路径 |
| child | array | 子资产列表 |
| child[].name | string | 资产名称 |
| child[].amount | number | 初始拥有数量 |
| child[].level | number | 初始等级 |
| child[].outcome | number | 单次产出数量 |
| child[].produce | string | 产出目标（Parent/上级资产名） |
| child[].time | number | 生产周期（秒） |
| child[].desc | string | 资产描述 |
| child[].cost | array | 购买消耗配置 |
| child[].cost[].name | string | 消耗品名称 |
| child[].cost[].quantity | number | 单次购买消耗数量 |
| child[].bonus | object | 五维加成配置 |
| child[].bonus.*.quantity | number | 加成倍率 |

## 9.5 持久化存储建议

如需实现数据存档，建议采用以下方案：

### 方案一：本地存储（LocalStorage）

```typescript
// 保存
localStorage.setItem('ea_save', JSON.stringify(userInfo));

// 读取
const saveData = JSON.parse(localStorage.getItem('ea_save'));
```

### 方案二：微信小程序存储

```typescript
// 保存
wx.setStorageSync('ea_save', userInfo);

// 读取
const saveData = wx.getStorageSync('ea_save');
```

### 方案三：服务器存档

```typescript
// 建议数据结构
interface SaveData {
  userId: string;
  version: string;
  timestamp: number;
  userInfo: UserInfo;
  assets: AssetConfig[];
}
```

## 9.6 数据迁移策略

当游戏版本更新导致数据结构变化时：

1. **版本号标记**：在存档数据中加入 `version` 字段
2. **迁移函数**：根据版本号执行对应的数据转换
3. **默认值填充**：新字段使用默认值

```typescript
function migrateSaveData(saveData: any): UserInfo {
  const version = saveData.version || '1.0.0';
  
  if (version === '1.0.0') {
    // 1.0.0 -> 1.1.0 的迁移逻辑
    saveData.newField = defaultValue;
  }
  
  saveData.version = '1.1.0';
  return saveData;
}
```
