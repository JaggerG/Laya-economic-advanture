# 系统架构

## 6.1 整体架构

Economic Adventure 采用典型的客户端游戏架构，基于 LayaAir 引擎构建：

```
+---------------------+
|     表现层（UI）      |
|  - Scene 场景管理     |
|  - UI 组件渲染        |
|  - 动画与特效         |
+---------------------+
|     逻辑层（Logic）   |
|  - 游戏主逻辑         |
|  - 资产生产计算       |
|  - 任务状态管理       |
|  - 卡牌系统逻辑       |
+---------------------+
|     数据层（Data）    |
|  - 配置数据           |
|  - 玩家存档           |
|  - 运行时状态         |
+---------------------+
|     引擎层（Engine）  |
|  - LayaAir 3.1.3     |
|  - 渲染引擎           |
|  - 事件系统           |
|  - 定时器系统         |
+---------------------+
|     平台层（Platform）|
|  - 微信小程序         |
|  - Web 浏览器         |
+---------------------+
```

## 6.3 数据流

### 6.3.1 资产生产数据流

```
Laya.timer (100ms)
    │
    ▼
AssetItem.changeProgress()
    │
    ├── 更新进度条 ──► UI 更新
    │
    └── 生产完成 ──► AssetItem.produce_goods()
                          │
                          ├── 计算产出数量（含加成）
                          ├── 暴击判定
                          ├── 更新 Data.Assets 数据
                          └── EventManager.Emit()
                                    │
                                    ├── updateParentLabel
                                    ├── xxx_COLLECT_update
                                    └── TaskItem.updateProgress()
```

### 6.3.2 购买资产数据流

```
用户点击购买按钮
    │
    ▼
AssetItem.handleBuyBtn()
    │
    ├── 扣除消耗资源
    │   ├── Parent 资源 ──► Data.Assets[panel].amount -= n
    │   ├── Employee 资源 ──► Data.Employee.amount -= n
    │   └── 上级资产 ──► bro.amount -= n
    │
    ├── 增加资产数量 ──► config.amount += n
    │
    └── 更新 UI
        ├── amount_label
        └── initData() ──► 重新计算生产速度
```

### 6.3.3 任务完成数据流

```
用户点击完成按钮
    │
    ▼
TaskItem.handleFinishClick()
    │
    ├── 抽奖逻辑
    │   ├── lottery.drawCard() ──► 随机抽取卡牌
    │   ├── lottery.collectCard() ──► 更新玩家卡牌数据
    │   └── lottery.drawSciencePoint() ──► 随机科学点数
    │
    ├── 更新玩家数据
    │   ├── userInfo.science_point += reward
    │   └── userInfo.Card[idx].has_amount += n
    │
    ├── 派发事件
    │   ├── Card_COLLECT_update
    │   ├── CardList_update
    │   ├── Science_update
    │   └── task_progress_update
    │
    └── 刷新任务
        └── TaskItem.initNextTask()
```
